Unity Video Game Development

Hello, my name is John and one of my Favorite hobbies to do whenever I have free time is to dabble in video game development within the Unity game engine. I have always loved video games, and the past few years in the Lewis Computer Science program have given me the skills to make small, usually bad games.

Unity is a free game engine that utilizes the idea of components, where components can be added, copied, and removed from other game elements and re-used on others. The components system also allows for certain properties to be edited from a visual editor in the "Inspector" window.

The main draw of Unity for me, however, is the fact that you can add a C# scripting component to any game object. The Unity API tied in with C# is actually very powerful, allowing you to edit most component properties from code. After getting familiar with Unity specific function calls, I've found that I really enjoy messing around with the Unity game engine and seeing some dumb ideas come to life.

John's Favorite Commands

Command Description
GameObject.Find(string) Finds a GameObject based off its name.
yield return new WaitForSeconds(int) Used in a Coroutine. Essentially a sleep function that's needlessly complex to implement.
UnityEngine.SceneManagement.SceneManager.GetActiveScene(int) Lets you know what current scene/level you have loaded.
